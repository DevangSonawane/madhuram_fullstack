0.  How to Use This Document
This document is a self-contained implementation brief for Codex (or any AI coding assistant). Every section describes exactly what to build, where to put it, and how it must work. Read it top-to-bottom and implement in the order presented.
The project is a React + Vite frontend for an Inventory Management System (IMS). The backend API is already deployed. Your job is purely frontend: adding new API calls, creating new pages, and updating existing pages.

Tech stack
• Framework: React 18 with React Router v6
• Build tool: Vite
• Styling: TailwindCSS + shadcn/ui component library
• State: Redux Toolkit (auth slice) + React Context (project, notification, theme)
• API layer: src/lib/api.js - a single export object with all fetch calls
• Auth: Bearer token stored in localStorage key "inventory_user" as JSON {token, user_id, ...}

File structure conventions
• All API methods live in src/lib/api.js - append new methods to the existing export object
• All pages live in src/pages/
• New routes are registered in src/App.jsx inside the existing <Routes> tree
• Reusable components live in src/components/
• shadcn/ui primitives are in src/components/ui/ - use them, do not re-invent

Critical rules
Never break existing functionality. Only add new code or extend existing functions. Do not remove or rename any existing api.js methods.
• Keep every fetch in api.js - no bare fetch() calls in pages or components
• All API calls must include getAuthHeaders() (Bearer token)
• Use handleResponse() for every response - it normalises errors and blobs
• Import components from @/components/ui/ using the path alias, not relative paths
• Use useToast() from @/hooks/use-toast for all success/error feedback


1.  Context - Business Flow & What Was Added
The IMS tracks materials from requisition to consumption. Every item in inventory follows this exact journey:

Sample  →  PR (Purchase Requisition)  →  PO (Purchase Order)  →  DC (Delivery Challan)  →  Inventory  →  Used in Sample / PR / MIR

Key design decisions in the backend
• DC creation auto-adds to inventory: When a Delivery Challan is created, all its items are automatically inserted into the Inventories table. No separate action is needed on the frontend.
• Stock deduction is automatic: When a PR item, Sample item, or MIR item includes inventory_id + issued_qty, the backend deducts stock automatically on save. No separate stock-out call is needed.
• Auto-match fallback: If inventory_id is not set, the backend provides POST endpoints to fuzzy-match items by name. The frontend should offer a "Link to Inventory" button for older records.
• Full audit trail: Every stock change is written to inventory_movements and inventory_history. The frontend needs to surface this data via dedicated pages.

searchInventoryTrace
Live search for the item-picker dropdown. Call this as the user types (debounce 300ms). Returns matching inventory items with available stock and a source chain label.
Query parameters:
• q (required): search keyword (item name or brand)
• project_id (optional): prioritises items from this project (shown first)
• min_qty (optional): only return items with available stock >= this value (default 0)
Each result includes source_chain_label - a human-readable string like: "DC: DC-2024-001 ← PO: PO-45 ← Sample: Block A". Show this in the dropdown so the user can confirm the correct item.
searchInventoryTrace: async ({ q, project_id, min_qty } = {}) => {
  const params = new URLSearchParams();

2.4  Inventory History - all 5 endpoints
getInventoryHistory
Returns paginated history across ALL inventory items. Supports filters, pagination, and sorting.
Parameters:
• inventory_id: filter to a specific item
• user_id: filter to changes by a specific user
• change_type: "stock_in" | "stock_out" | "adjustment" | "created" | "updated" | "deleted"
• source_type: "dc" | "po" | "pr" | "sample" | "mir" | "manual"
• project_id: filter to a specific project
• from / to: ISO date strings (YYYY-MM-DD)
• page / limit: pagination (default page=1, limit=20, max=100)
• sort: "asc" | "desc" (default desc)
getInventoryHistory: async (filters = {}) => {
  const params = new URLSearchParams();
  const allowed = ["inventory_id","user_id","change_type","source_type","project_id","from","to","page","limit","sort"];
  allowed.forEach((k) => { if (filters[k] != null && filters[k] !== "") params.set(k, String(filters[k])); });
  const query = params.toString();
  const response = await fetch(`${BASE_URL}/api/inventory-history${query ? "?" + query : ""}`, {
    headers: getAuthHeaders(),
  });
  return handleResponse(response);
},

3.  Shared Component - InventoryPicker
This is the most important new component. It is used inside the PR create form, Sample create/edit form, and MIR create form. Build it once and import it everywhere.
File: src/components/InventoryPicker.jsx

3.1  What it does
InventoryPicker is an inline search field that appears on each item row inside a form. As the user types a material description, it calls /api/inventory-trace/search (debounced 300ms) and shows a dropdown of matching inventory items. When the user selects an item, the component calls an onSelect callback with the chosen inventory_id and available quantity.

3.2  Props
PropTypeDescriptionproject_idnumber | nullCurrent project ID - passed to the search endpoint to prioritise matching itemsinitialValuestringPre-fill the search input (e.g. the item description already typed in the row)selectedIdnumber | nullCurrently linked inventory_id - highlights the selected item in the listonSelectfunctionCalled with ({ inventory_id, name, brand, available_qty, units, source_chain_label }) when user picks a rowonClearfunctionCalled when the user removes the selection (X button)disabledbooleanDisables the field (e.g. when form is in view-only mode)minQtynumberOnly show items with available_qty >= minQty (default 0)
3.3  Behaviour
• Debounce the search call by 300ms after the last keystroke
• Show a loading spinner inside the dropdown while the request is in flight
• Each dropdown row shows: item name (bold), brand (muted), available quantity + units, and source_chain_label on a second line
• If no results, show "No inventory items found for this search"
• Once an item is selected: collapse the dropdown, show a green confirmation chip inside the field with the item name and available qty, and render a small X button to clear it
• If selectedId is set on mount, fetch the item details and show the chip immediately without opening the dropdown
• The chip text: "{item name} - {available_qty} {units} available"
• Source chain label appears below the chip in muted text: "DC: DC-2024-001 ← PO-45 ← Sample: Block A"

3.4  Skeleton implementation
import { useState, useRef, useCallback } from "react";
import { api } from "@/lib/api";
import { Input } from "@/components/ui/input";
import { Badge } from "@/components/ui/badge";
import { X, Search, Loader2, CheckCircle2 } from "lucide-react";

export default function InventoryPicker({ project_id, initialValue = "", selectedId,
                                          onSelect, onClear, disabled, minQty = 0 }) {
  const [query, setQuery]     = useState(initialValue);
  const [results, setResults] = useState([]);
  const [loading, setLoading] = useState(false);
  const [open, setOpen]       = useState(false);
  const [selected, setSelected] = useState(null);
  const timerRef = useRef(null);

  const search = useCallback(async (q) => {
    if (!q || q.length < 2) { setResults([]); setOpen(false); return; }
    setLoading(true);
    const res = await api.searchInventoryTrace({ q, project_id, min_qty: minQty });
    setLoading(false);
    if (res.success && Array.isArray(res.data)) {
      setResults(res.data);
      setOpen(true);
    }
  }, [project_id, minQty]);

  const handleChange = (e) => {
    const v = e.target.value;
    setQuery(v);
    clearTimeout(timerRef.current);
    timerRef.current = setTimeout(() => search(v), 300);
  };

  const handlePick = (item) => {
    setSelected(item);
    setOpen(false);
    setQuery("");
    onSelect?.(item);
  };

  const handleClear = () => {
    setSelected(null);
    setQuery("");
    onClear?.();
  };

  // Render chip when item is selected
  if (selected) return (
    <div className="flex flex-col gap-1">
      <div className="flex items-center gap-2 p-2 border rounded-md bg-green-50 border-green-300">
        <CheckCircle2 className="h-4 w-4 text-green-600 shrink-0" />
        <span className="text-sm font-medium flex-1">
          {selected.name} - {selected.available_qty} {selected.units} available
        </span>
        {!disabled && <button onClick={handleClear}><X className="h-4 w-4 text-gray-400" /></button>}
      </div>
      {selected.source_chain_label && (
        <span className="text-xs text-muted-foreground pl-2">{selected.source_chain_label}</span>
      )}
    </div>
  );

  return (
    <div className="relative">
      <div className="relative">
        <Search className="absolute left-2.5 top-2.5 h-4 w-4 text-muted-foreground" />
        <Input className="pl-8" placeholder="Search inventory..." value={query}
               onChange={handleChange} disabled={disabled} />
        {loading && <Loader2 className="absolute right-2.5 top-2.5 h-4 w-4 animate-spin text-muted-foreground" />}
      </div>
      {open && results.length > 0 && (
        <div className="absolute z-50 w-full mt-1 bg-white border rounded-md shadow-lg max-h-64 overflow-y-auto">
          {results.map((item) => (
            <button key={item.inventory_id} className="w-full text-left p-3 hover:bg-gray-50 border-b last:border-0"
              onClick={() => handlePick(item)}>
              <div className="flex items-center justify-between">
                <span className="font-medium text-sm">{item.name}</span>
                <Badge variant="outline">{item.available_qty} {item.units}</Badge>
              </div>
              <div className="text-xs text-muted-foreground mt-0.5">{item.brand}</div>
              {item.source_chain_label && (
                <div className="text-xs text-blue-600 mt-0.5">{item.source_chain_label}</div>
              )}
            </button>
          ))}
        </div>
      )}
      {open && results.length === 0 && !loading && (
        <div className="absolute z-50 w-full mt-1 bg-white border rounded-md shadow-sm p-3 text-sm text-muted-foreground">
          No inventory items found for this search.
        </div>
      )}
    </div>
  );
}

export default function MyPage() {
  const { selectedProject } = useProject();
  const { user }            = useAuth();
  const { toast }           = useToast();
  const projectId = selectedProject?.id ?? selectedProject?.project_id ?? null;

  const [data, setData]       = useState([]);
  const [loading, setLoading] = useState(false);
  const [error, setError]     = useState(null);

  useEffect(() => { fetchData(); }, [projectId]);

  const fetchData = async () => {
    setLoading(true);
    const res = await api.someMethod({ project_id: projectId });
    setLoading(false);
    if (res.success) { setData(res.data); }
    else { setError(res.error); toast({ title: "Error", description: res.error, variant: "destructive" }); }
  };

  if (loading) return <Skeleton className="h-48 w-full" />;
  if (error)   return <div className="text-destructive">{error}</div>;
  return ( /* your JSX */ );
}
